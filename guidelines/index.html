[DRAFT]

  1. Hereinafter member functions are called methods, and data members — fields, even if in the standard they aren’t. Deal with it.
  2. Use tabs for indentation, spaces for alignment, like this: [picture].
  3. Private and protected methods, as well as free functions, are named using snake_case.
  4. Classes and public methods are named using UpperCamelCase (as opposed to lowerCamelCase).
  5. Never use operators new and delete explicitly. Prefer standard containers or smart pointers.
